# recall — 기록 회상·검색

## 언제 쓰나
대표가 과거 기록을 물을 때. "지난주 B고객이랑 뭐 얘기했지?", "A업체 관련 찾아줘",
"그 아이디어 뭐였지?". 폴더를 뒤지지 않고 말로 물어보게 하는 기능.

## 읽는 규약 (기준: `notes/README.md` — `quick-note` 와 완전히 동일)
검색 대상 폴더 5종. 경로 표기도 `quick-note` 와 동일하다:

| 폴더 | 들어 있는 것 |
|---|---|
| `notes/inbox/` | 분류 애매한 기본함 |
| `notes/meetings/` | 미팅·통화·상담 기록 |
| `notes/customers/` | 고객·거래처별 기록 |
| `notes/ideas/` | 아이디어·메모 |
| `notes/issues/` | 이슈·문제 |

파일 경로 표기: `notes/<폴더>/YYYY-MM-DD-짧은제목.md` (quick-note가 저장한 그 위치를 읽는다).

## 동작 순서
1. **질의 해석** — 키워드(고객명·업체명·주제)와 기간(지난주·어제·특정일)을 뽑는다.
2. **검색** — `notes/` 전체를 훑는다. 질의 성격에 따라 폴더 우선순위를 둔다
   (고객명 → `customers/`, 미팅 → `meetings/`, 이슈 → `issues/`). 파일명 날짜로 기간을 좁힌다.
3. **요약 + 출처** — 찾은 내용을 짧게 정리하고 **어느 파일에서 나왔는지** 경로를 밝힌다.
4. **없으면** — "관련 기록을 못 찾았어요"라고 알리고, 비슷한 키워드나 기간을 다시 물어본다.
5. **상태 갱신** — `state/status.json` 의 `today.activities` 에 `{time,type:"note",summary:"회상: …"}`
   추가, `lastActiveAt`·`updatedAt` 갱신 후 `state/dashboard-data.js` 재생성
   (`docs/STATE_CONTRACT.md` 준수). (회상은 별도 카운트를 올리지 않는다.)

## 출력 형식 예시
```
[B고객 관련 기록]
• 7/15 코칭 — 재계약 의사 있음, 가격은 부담. (notes/customers/2026-07-15-B고객-코칭.md)
• 7/18 통화 — 다음 시즌 물량 문의. (notes/customers/2026-07-18-B고객-통화.md)

정리: 재계약에 긍정적이나 가격이 관건. 물량 확대 여지 있음.
```
못 찾은 경우:
```
'C업체' 관련 기록은 아직 없어요. 다른 이름으로 저장됐을 수도 있는데, 힌트 주시면 다시 찾아볼게요.
```
