# quick-note — 빠른 기록

## 언제 쓰나
대표가 나중에 쓸 정보를 던질 때. "방금 A업체가 단가 조정 요청함", "메모: …", "이거 적어둬".
머리를 비우게 하는 가장 일상적인 기능.

## 저장 규약 (기준: `notes/README.md`)
분류 폴더 5종 — `recall` 스킬과 **완전히 동일**하다:

| 폴더 | 넣는 것 |
|---|---|
| `notes/inbox/` | 분류 애매한 기본함 |
| `notes/meetings/` | 미팅·통화·상담 기록 |
| `notes/customers/` | 고객·거래처별 기록 |
| `notes/ideas/` | 아이디어·메모 |
| `notes/issues/` | 이슈·문제 (issue-tracker 연계) |

파일 경로 표기: `notes/<폴더>/YYYY-MM-DD-짧은제목.md`
(예: `notes/customers/2026-07-20-A업체-단가조정.md`)

## 동작 순서
1. **분류 판단** — 내용을 보고 위 5폴더 중 하나 선택. 애매하면 `inbox/`.
   **경계 규칙:** 특정 고객·거래처가 주인공이면 — 첫 문의·상담 통화라도 — `customers/` 우선.
   `meetings/` 는 다자 회의·사건 중심 기록용. (기준: `notes/README.md`)
2. **파일 결정 — 저장 전 중복 검색(의무)** — 먼저 해당 폴더에서 같은 주제의 기존 파일을 찾는다
   (제목·태그·핵심어 기준). **날짜가 달라도 같은 주제면 새 파일을 만들지 말고 기존 파일에
   "## 추가 (YYYY-MM-DD)" 섹션으로 이어 적는다** (파일명 날짜는 최초 기록일 유지). 기존 파일이
   없을 때만 `notes/<폴더>/YYYY-MM-DD-짧은제목.md` 로 새로 만든다. 같은 주제 메모가 파일 여러 개로
   흩어지면 회상이 반쪽이 된다 — 이 검색은 생략하지 않는다.
3. **저장** — 아래 형식으로 기록. 던진 원문을 보존하고 한 줄 요약을 붙인다.
4. **짧게 확인 응답** — 어디에 저장했는지 1줄.
5. **상태 갱신** — `state/status.json` 의 `today.counts.notes` +1, `today.activities` 에
   `{time,type:"note",summary}` 추가, `updatedAt`·`lastActiveAt` 갱신 후 `state/dashboard-data.js`
   재생성 (`docs/STATE_CONTRACT.md` 준수).

## 저장 파일 형식
```markdown
# A업체 단가조정

- 날짜: 2026-07-20 14:30
- 분류: customers
- 태그: A업체, 단가

방금 미팅에서 A업체가 단가 조정 요청함. (원문)
요약: A업체 단가 재협의 필요 — 다음 미팅 전 견적 근거 준비.
```

## 응답 예시 (텔레그램)
```
기록했어요 → notes/customers/2026-07-20-A업체-단가조정.md
(고객 기록으로 분류. '지난 A업체 얘기' 물어보면 찾아드릴게요.)
```
