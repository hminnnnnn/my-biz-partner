# reminder — 스마트 리마인더

## 언제 쓰나
- 대표가 맡길 때: "내일 미팅 전에 견적서 확인 알려줘", "금요일 세금신고 챙겨줘".
- 파트너가 먼저: 일정·이슈를 브리핑할 때 "미팅 전에 견적서 확인이 필요해 보여요"처럼 선제 제안.

## 동작 순서
1. **대상·시점 파악** — 무엇을, 언제(어느 일정 전 / 특정 날짜)까지.
2. **기록** — 리마인더를 `notes/inbox/` 에 저장한다(분류 규약의 기본함). 파일명
   `notes/inbox/YYYY-MM-DD-리마인더-짧은제목.md`. 파일에 **기한(due)** 을 명시한다.
3. **브리핑 연동** — `daily-briefing` 이 실행될 때 아직 안 끝난 리마인더 중 기한이 임박한 것을
   함께 알린다. (리마인더는 이렇게 브리핑·세션 시작 때 표면화된다.)
4. **상태 갱신** — `state/status.json` 의 `today.counts.reminders` +1, `today.activities` 에
   `{time,type:"reminder",summary}` 추가, `updatedAt`·`lastActiveAt` 갱신 후
   `state/dashboard-data.js` 재생성 (`docs/STATE_CONTRACT.md` 준수).

## 저장 파일 형식
```markdown
# 리마인더 — 견적서 확인

- 등록: 2026-07-20 17:00
- 기한(due): 2026-07-21 10:00 (A업체 미팅 전)
- 상태: 대기

내일 A업체 미팅 전에 견적서 확인.
```

## 중요한 한계 (정직하게 안내)
이 비서는 **노트북이 켜져 있고 받는 창이 살아 있을 때만** 동작한다. 정해진 시각에 **자동으로
알림을 울리는 상시 스케줄러는 이 버전에 없다.** 그래서 리마인더는:
- 세션을 시작할 때, 그리고 브리핑을 요청할 때 **파트너가 확인해 알려주는** 방식으로 동작한다.
- 정확한 시각 푸시가 꼭 필요하면 지금은 휴대폰 기본 알람과 병행하도록 안내한다.
(상시 구동 자동 알림은 다음 단계 로드맵.)

## 응답 예시 (텔레그램)
```
리마인더 등록했어요 → 내일 10:00 A업체 미팅 전 '견적서 확인'.
내일 아침 브리핑 때 다시 짚어드릴게요. (정시 알림이 꼭 필요하면 폰 알람도 같이 맞춰두세요.)
```
