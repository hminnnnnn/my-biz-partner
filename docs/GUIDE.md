# 사용 가이드 — 내 비즈니스 파트너

텔레그램으로 대화하는, **내 사업을 아는 전담 AI 비서**의 사용법입니다.
개발 지식은 필요 없습니다. 처음 30분만 따라 하면, 그다음부터는 메시지 한 문장으로 일이 끝납니다.

> 이 문서는 언제든 다시 펼쳐 보는 사전입니다. 처음이라면 **1 → 2절**만 읽고 바로 시작하세요.
> 각 기능이 궁금해질 때 **3절**을, 막히면 **5절**을 찾아 오면 됩니다.

---

## 1. 한눈에 보기 — 하루가 이렇게 흐릅니다

이 비서는 "묻고 시키는" 도구가 아니라, **내 사업 맥락을 기억하는 동료**에 가깝습니다.
캘린더·기록·초안을 각각 다른 앱에서 관리하던 걸, **텔레그램 한 곳에서 말로** 처리합니다.

| 때 | 이렇게 말하면 | 벌어지는 일 |
|---|---|---|
| 아침 | "내일 일정 브리핑해줘" | 일정 + 미팅 전 챙길 것을 1분 브리핑 |
| 수시 | "방금 A업체가 단가 조정 요청함" | 알아서 분류해 기록. 머리를 비웁니다 |
| 필요할 때 | "지난주 B고객이랑 뭐 얘기했지?" | 흩어진 기록을 찾아 요약 |
| 응대할 때 | "이 문의에 답장 초안 잡아줘" | 내 말투로 초안을 뽑아 줌 |
| 큰 일 | "신제품 런칭 기획 진행해줘" | 여러 역할의 팀을 꾸려 단계별로 진행 |

**하루 흐름 한 줄:** 아침 브리핑 → 수시로 툭툭 기록 → 필요할 때 회상·초안 → 큰 일은 팀 모드.
쌓인 활동은 **대시보드**(`dashboard/index.html`) 화면에서 한눈에 보입니다.

## 2. 처음 30분 셋업 여정

> 🧑‍🏫 **가장 쉬운 길 — 코치를 부르세요.** 이 폴더에서 터미널에 `claude` 를 켜고 **"셋업 시작"**
> 이라고만 하면, 셋업 코치가 지금 어디까지 됐는지 진단하고 **텔레그램 연결·정체성·캘린더·첫 대화**를
> 순서대로 한 번에 하나씩 안내하며 매 단계 진짜 됐는지 확인해 줍니다. 세션이 꺼져도 다시
> "셋업 시작"이라고 하면 멈춘 지점부터 이어집니다. **아래 여섯 단계는 코치가 안 될 때 참고하는
> 지도**이니, 코치가 이끄는 대로 따라가면 아래를 직접 읽지 않아도 됩니다.

여섯 단계입니다. 각 단계마다 **"이게 보이면 성공"**을 적어 뒀으니, 그것만 확인하고 넘어가세요.

### 단계 1 · 설치 (터미널 명령 2개)

내려받아 압축 푼 폴더로 이동한 뒤, 설치 스크립트를 실행합니다.

```
cd ~/Downloads/my-biz-partner
./install.sh
```

> **이게 보이면 성공** — 화면 마지막에 `[3/3] 완료!` 와 함께 다음 안내 상자가 뜹니다:
> `1) 텔레그램 연결` `2) 나를 소개하기` `3) 캘린더 연결(선택)` `4) 대시보드 열기`.
> 중간에 `✓ macOS 확인됨`, `✓ state/dashboard-data.js 갱신 완료` 도 지나갑니다.
> (여러 번 실행해도 안전합니다. 기존 기록은 절대 지우지 않습니다.)

### 단계 2 · 텔레그램 연결

`docs/SETUP_TELEGRAM.md` 를 따라 합니다. 핵심은 **터미널 창이 두 개**라는 것뿐입니다 —
봇 토큰을 등록하는 **설정 창**, 메시지를 받는 **받는 창**(열어 둬야 연결 유지).

1. 텔레그램 **@BotFather** 에서 `/newbot` 으로 봇을 만들고 토큰을 받습니다.
2. 설정 창에서 `/telegram:configure` 로 토큰을 등록합니다.
3. 새 터미널 창(=받는 창)에서 `claude --channels plugin:telegram@claude-plugins-official` 실행 — **닫지 마세요.**
4. 휴대폰에서 봇에게 메시지를 보내 페어링(짝 맺기)합니다. 승인은 **설정 창에서 직접** 합니다.

> **이게 보이면 성공** — 휴대폰에서 봇에게 **"안녕, 내 사업 알고 있어?"** 라고 보냈을 때
> 비서가 답을 하면 연결된 것입니다. (아직 정체성을 안 채웠으면 "같이 채워보자"고 합니다.)

### 단계 3 · 나를 소개하기 (정체성 채우기)

`identity/IDENTITY.md` 의 **4칸**(내 사업 / 내 고객 / 내 말투·톤 / 지금의 우선순위)만 채우면 됩니다.
직접 타이핑이 막막하면, 텔레그램에 **"정체성 같이 채워줘"** 라고 하면 파트너가 하나씩 물어보며 채워 줍니다.
감이 안 오면 `identity/EXAMPLE.md` 를 열어 보세요.

> **이게 보이면 성공** — 대시보드 "에이전트 상태" 카드에 **정체성 로드됨** 뱃지가 뜹니다.
> (안 채웠으면 `정체성 미입력 — identity/IDENTITY.md 를 채워 주세요` 로 보입니다.)

### 단계 4 · 첫 대화

텔레그램에 아무 말이나 걸어 보세요. 내 사업 맥락이 반영된 답이 오는지 봅니다.

```
나: 안녕, 내 사업 알고 있어?
나: 방금 A업체가 단가 조정 요청했어
나: 내일 일정 브리핑해줘
```

> **이게 보이면 성공** — 두 번째 문장에서 파트너가 "기록했어요"라고 답하고, 대시보드
> "오늘 활동"의 **기록** 숫자가 올라갑니다. 범용 답이 아니라 내 고객·업종에 맞춘 톤이면 잘 된 것입니다.

### 단계 5 · 캘린더 연결 (선택 — 폴백 있음)

`docs/SETUP_CALENDAR.md` 참고. 구글 캘린더 연동이 어려우면 건너뛰어도 됩니다.
대신 `notes/calendar-fallback.md` 파일에 이번 주 일정을 아무 형식으로나 적어 두면, 브리핑이 그 파일을 읽습니다.

> **이게 보이면 성공** — "내일 일정 브리핑해줘" 라고 했을 때, 캘린더(연동 시) 또는 폴백 파일의
> 일정으로 답하면 성공. 둘 다 비어 있으면 "등록된 일정이 없어요" 안내가 옵니다.

### 단계 6 · 대시보드 열기

`dashboard/index.html` 을 **더블클릭**하면 브라우저에 내 비서 상태 화면이 열립니다. 서버·설치 불필요.

> **이게 보이면 성공** — 상단에 **"내 비즈니스 파트너 · 대시보드"** 제목과 함께 세 개의 카드
> (에이전트 상태 / 오늘 활동 / 프로젝트 팀 진행현황)가 보입니다. 화면은 5초마다 자동 새로고침됩니다.

## 3. 기능 사전 — 스킬 7종

모든 기능은 **자동으로** 골라 쓰입니다. 아래 예시처럼 편하게 말하면 파트너가 알맞은 기능을 실행합니다.
저장 경로는 실제 파일 위치이니, 나중에 직접 열어 봐도 됩니다.

### 3-1. daily-briefing · 일정 브리핑 + 미팅 준비

- **언제 쓰나** — 아침에, 또는 미팅 앞두고. 일정과 "미팅 전 챙길 것"을 짧게 정리받고 싶을 때.
- **이렇게 말하세요** — "내일 일정 브리핑해줘" / "오늘 뭐 있지?" / "이번 주 일정 알려줘" / "15시 미팅 사전 준비 챙겨줘"
- **무슨 일이 일어나나** — 캘린더(연동 시) 또는 `notes/calendar-fallback.md` 를 읽고, 미팅 상대 관련 기록을 `notes/` 에서 훑어 붙입니다. 파일을 새로 만들진 않고, 브리핑 횟수만 대시보드에 +1.

```
나: 내일 일정 브리핑해줘
파트너:
[내일(7/21) 브리핑]
• 10:00 A업체 미팅 — 단가 재협의
  ↳ 지난주 A업체가 단가 조정 요청함(기록 있음). 견적 근거 챙기세요.
• 15:00 B고객 코칭
• 저녁 일정 없음

준비 팁: 오전 미팅 전에 견적서 확인이 필요해 보여요.
```

### 3-2. quick-note · 빠른 기록

- **언제 쓰나** — 나중에 쓸 정보를 툭 던질 때. 가장 일상적으로 쓰는 기능입니다.
- **이렇게 말하세요** — "방금 미팅에서 A업체가 단가 조정 요청함" / "메모: 여름 프로모션 아이디어" / "이거 기록해줘" / "까먹기 전에 적어둬"
- **무슨 일이 일어나나** — 내용을 보고 5개 폴더 중 하나로 분류해 `notes/<폴더>/YYYY-MM-DD-짧은제목.md` 에 저장합니다. (특정 고객이 주인공이면 `customers/` 우선.)

```
나: 방금 미팅에서 A업체가 단가 조정 요청함
파트너:
기록했어요 → notes/customers/2026-07-20-A업체-단가조정.md
(고객 기록으로 분류. '지난 A업체 얘기' 물어보면 찾아드릴게요.)
```

### 3-3. recall · 기록 회상·검색

- **언제 쓰나** — 예전에 던진 내용을 다시 찾을 때. 폴더를 뒤지지 말고 말로 물어보세요.
- **이렇게 말하세요** — "지난주에 B고객이랑 뭐 얘기했지?" / "A업체 관련 기록 찾아줘" / "그때 그 아이디어 뭐였지?" / "반품 이슈 어떻게 됐더라?"
- **무슨 일이 일어나나** — `notes/` 전체(`projects/` 포함)를 훑어 요약하고, **어느 파일에서 나왔는지** 경로까지 알려 줍니다. (읽기만 하므로 새 파일은 생기지 않습니다.)

```
나: 지난주에 B고객이랑 뭐 얘기했지?
파트너:
[B고객 관련 기록]
• 7/15 코칭 — 재계약 의사 있음, 가격은 부담. (notes/customers/2026-07-15-B고객-코칭.md)
• 7/18 통화 — 다음 시즌 물량 문의. (notes/customers/2026-07-18-B고객-통화.md)

정리: 재계약에 긍정적이나 가격이 관건. 물량 확대 여지 있음.
```

### 3-4. draft-writer · 내 말투로 초안 쓰기

- **언제 쓰나** — 응대·공지·메일을 내 톤으로 빠르게 뽑고 싶을 때. 시간 절약이 바로 체감됩니다.
- **이렇게 말하세요** — "이 문의에 답장 초안 잡아줘" / "공지 초안 써줘" / "이 메일 답장 좀" / "거절 메시지 부드럽게 써줘"
- **무슨 일이 일어나나** — `identity/IDENTITY.md` 의 말투를 반영해 초안을 만들고, 나중에 다시 찾을 수 있게 대상 고객의 `notes/customers/` 파일에 "## 초안 (날짜)" 로 이어 적습니다(대상이 불특정하면 `notes/inbox/`). **보낼지 말지는 대표 몫** — 파트너는 확인만 구합니다.

```
나: 이 단가 문의에 답장 초안 잡아줘
파트너:
[답장 초안 — 정중한 존댓말]
안녕하세요, 대표님. 문의 주셔서 감사합니다.
말씀하신 단가 건은 내부에서 검토 후 이번 주 안에 회신드리겠습니다.
급하신 부분이 있으면 편하게 말씀해 주세요. 감사합니다.

— 이대로 보낼까요? 톤이나 내용 고칠 곳 있으면 알려주세요.
```

### 3-5. issue-tracker · 이슈 통합관리

- **언제 쓰나** — 문제가 터졌을 때. 한 줄로 던지면 기록하고, 관련된 지난 기록까지 묶어 상태를 추적합니다.
- **이렇게 말하세요** — "방금 CS에서 반품 이슈 터졌어, 기록하고 관련된 것 묶어줘" / "이 문제 이슈로 잡아줘" / "지난번 그 클레임이랑 연결해줘"
- **무슨 일이 일어나나** — `notes/issues/YYYY-MM-DD-짧은제목.md` 에 저장하고, 같은 고객·주제의 기존 기록을 찾아 파일 안 "관련" 항목에 연결합니다. 지금 상태(열림/진행/해결)와 다음 할 일을 함께 제안합니다.

```
나: 방금 CS에서 반품 이슈 터졌어, 기록하고 관련된 것 묶어줘
파트너:
이슈 잡았어요 → notes/issues/2026-07-20-반품-클레임.md (상태: 열림)
연결: 7/12 A고객 주문 기록과 묶었어요.
다음 할 일: 교환/환불 방침 정하고 오늘 중 회신 → 정해지면 알려주세요, 초안 잡아드릴게요.
```

### 3-6. reminder · 스마트 리마인더

- **언제 쓰나** — 나중에 챙길 일을 맡길 때. 파트너가 브리핑 중에 먼저 "미팅 전에 ○○ 확인하셔야 해요"라고 짚어 주기도 합니다.
- **이렇게 말하세요** — "내일 미팅 전에 견적서 확인 알려줘" / "금요일에 세금신고 챙겨줘" / "이거 잊지 않게 리마인드"
- **무슨 일이 일어나나** — `notes/inbox/YYYY-MM-DD-리마인더-짧은제목.md` 에 기한(due)과 함께 저장하고, 이후 **세션을 시작할 때·브리핑할 때** 파트너가 표면화해 알려 줍니다.

```
나: 내일 미팅 전에 견적서 확인 알려줘
파트너:
리마인더 등록했어요 → 내일 10:00 A업체 미팅 전 '견적서 확인'.
내일 아침 브리핑 때 다시 짚어드릴게요. (정시 알림이 꼭 필요하면 폰 알람도 같이 맞춰두세요.)
```

> 정해진 시각에 **자동으로 알림을 울리는 상시 스케줄러는 이 버전에 없습니다.** 정시 푸시가 꼭
> 필요하면 휴대폰 기본 알람과 병행하세요. (상시 구동 자동 알림은 다음 단계 로드맵입니다.)

### 3-7. project-team · 프로젝트 팀 모드

- **언제 쓰나** — **여러 단계·여러 산출물·여러 역할**이 필요한 큰 일에만. 산출물 하나로 한 번에 끝나는 일은 위의 기본 기능으로 충분합니다.
- **이렇게 말하세요** — "다음 달 신제품 런칭 기획 진행해줘" / "새 서비스 출시 준비 시작하자" / "이거 프로젝트로 크게 진행해줘"
- **무슨 일이 일어나나** — PM·리서치·디자인·개발 같은 역할의 팀을 꾸리고, 진행현황을 `state/team.json` 에 기록해 **대시보드에 실시간으로** 보여 줍니다. 결과물은 `notes/projects/<YYYY-MM-프로젝트슬러그>/` 에 단계별로 저장됩니다. (업종에 맞게 역할명은 바뀝니다 — 코칭이면 `개발`→`운영` 등.)

```
나: 다음 달 신제품 런칭 기획 진행해줘
파트너:
'신제품 런칭 기획' 팀 꾸렸어요. 대시보드에서 진행현황 보실 수 있어요.
• PM: 일정·범위 정리 (완료)
• 리서치: 경쟁사·시장 조사 (진행 중)
• 디자인/개발: 대기
리서치 끝나면 시안 단계로 넘어가면서 다시 알려드릴게요.
```

> **뉴스 인사이트 브리핑**(매일 내 업종 관련 소식을 요약)은 이 버전에는 없습니다 —
> **온라인 세션에서 확장 예정**입니다.

## 4. 내 데이터가 쌓이는 구조

모든 기록은 내 컴퓨터의 이 폴더 안, `notes/` 아래에 **분류되어** 쌓입니다. 규약의 기준 문서는 `notes/README.md` 입니다.

| 폴더 | 무엇이 들어가나 |
|---|---|
| `notes/inbox/` | 분류가 애매한 기본함 (리마인더도 여기) |
| `notes/meetings/` | 여러 사람이 얽힌 회의·사건 중심 기록 |
| `notes/customers/` | 고객·거래처별 기록 (특정 고객이 주인공이면 여기 우선) |
| `notes/ideas/` | 아이디어·떠오른 생각 |
| `notes/issues/` | 이슈·문제 (issue-tracker 연계) |
| `notes/projects/` | 팀 모드 산출물 전용 (`<YYYY-MM-슬러그>/01-기획.md` …) |

파일명은 항상 `YYYY-MM-DD-짧은제목.md`. 같은 날 같은 주제는 새 파일 대신 **이어 적습니다.**

**상태 파일과 대시보드의 관계** — 파트너가 일을 할 때마다 `state/` 안 두 파일에 활동을 남깁니다:

- `state/status.json` — 오늘 한 일·횟수 (기본 기능이 씀)
- `state/team.json` — 팀 프로젝트 진행현황 (팀 모드가 씀)
- `state/dashboard-data.js` — 위 둘을 합친 **파생 파일**. 대시보드는 이 파일 **하나만** 읽습니다.

내가 이 파일들을 직접 만질 일은 없습니다. 그냥 **말로 시키면 파트너가 갱신 → 대시보드에 반영**되는
한 방향 흐름이라고만 알아 두면 됩니다.

## 5. 자주 막히는 곳 & 해결

| 증상 | 원인 · 해결 |
|---|---|
| "일정이 없다"고만 해요 | 캘린더 미연동 + 폴백 비어 있음. `notes/calendar-fallback.md` 에 이번 주 일정을 적어 두세요. 아무 형식이나 됩니다. |
| 봇이 응답이 없어요 | 십중팔구 **받는 창이 꺼진** 것. 새 터미널에서 `claude --channels plugin:telegram@claude-plugins-official` 를 다시 실행하세요. |
| 봇이 페어링 코드를 안 줘요 | 받는 창이 켜져 있는지 확인 후, 봇에게 메시지를 다시 보내세요. |
| 그래도 응답이 없어요 | 토큰 문제일 수 있어요. 설정 창에서 `/telegram:configure` 로 다시 등록하세요. |
| 정체성을 잘못 적었어요 | `identity/IDENTITY.md` 를 열어 직접 고치거나, 텔레그램에 "정체성 다시 정리하자"고 하면 파트너가 물어보며 고쳐 줍니다. |
| 대시보드가 안 바뀌어요 | 5초마다 자동 새로고침됩니다. 그래도 낡았으면 브라우저 새로고침(⌘R). |

> **90%의 문제는 "받는 창이 꺼져 있어서"** 입니다. 뭔가 안 되면 그 창부터 확인하세요.

## 6. 알아 둘 한계 (정직하게)

- **노트북을 켜 두고, 받는 창을 열어 둔 동안만 작동합니다.** 그 창을 닫거나 노트북을 끄면 비서도
  잠들고, 그 사이 보낸 메시지는 유실될 수 있습니다. (다시 켜면 이어서 대화할 수 있어요.)
  24시간 상시 구동은 이 버전의 범위가 아니라 **다음 단계 로드맵**입니다.
- **텔레그램 연동은 리서치 프리뷰(실험 단계)** 기능입니다. 가끔 응답이 늦거나 재연결이 필요할 수
  있고, 명령어가 예고 없이 바뀔 수 있습니다. 워크숍 직전에 최신 공식 문서를 한 번 더 확인하세요.
- **내 데이터는 내 컴퓨터에 있습니다.** 모든 기록은 이 폴더(`notes/`) 안에 저장되며, 외부로
  내보내지 않습니다.

## 7. 더 잘 쓰는 팁

- **툭툭 던지는 습관이 자산이 됩니다.** 미팅 직후 "방금 …" 한 줄이면 충분합니다. 완결된 문장이
  아니어도 됩니다. 이 조각들이 나중에 회상·브리핑·초안의 재료가 됩니다.
- **말투는 대화로 학습됩니다.** `IDENTITY.md` 의 톤을 기본으로 쓰되, 초안이 어색하면 "더 친근하게",
  "더 짧게"처럼 고쳐 달라고 하세요. 자주 쓰는 호칭·인사말을 정체성에 적어 두면 더 내 톤에 가까워집니다.
- **팀 모드는 아껴 쓰세요.** 판단 기준: 산출물이 **하나**이고 한 번에 끝나면 → 기본 기능(단독 처리).
  단계가 갈리고 산출물이 **여러 개**(리서치 + 기획서 + 시안 …)면 → 팀 모드. 애매하면 파트너가
  "팀을 꾸릴까요, 제가 바로 정리할까요?"라고 물어봅니다.
- **아침 1분 브리핑을 루틴으로.** 하루를 "내일 일정 브리핑해줘"로 열면, 리마인더와 미팅 준비가
  자동으로 표면화됩니다.
