# 텔레그램 연결하기

> 📄 **이 문서는 수동 진행용 백업입니다.** 권장 경로는 터미널에서 `claude` 를 켜고 **"셋업 시작"**
> (또는 "텔레그램 연결") 이라고 말해 코치가 한 단계씩 안내받는 것입니다. 코치가 안 될 때 이 문서를 지도로 쓰세요.

휴대폰에서 텔레그램으로 내 비서에게 말을 걸 수 있게 만드는 단계입니다.
Claude Code의 **Channels(채널)** 기능을 씁니다. 개발 지식은 필요 없고, 아래 순서대로만 하면 됩니다.

> ⚠️ **먼저 읽어 주세요.** Channels는 **리서치 프리뷰**(실험 단계) 기능입니다. 명령어나 문법이
> 예고 없이 바뀔 수 있어요. **워크숍 직전에 최신 공식 문서를 한 번 더 확인**하세요:
> Claude Code 문서의 Channels 항목. 이 가이드와 화면이 다르면 최신 문서를 우선하세요.

---

## 큰 그림 — 창이 두 개예요

헷갈리는 유일한 이유는 터미널 창이 **두 개**라는 점입니다. 먼저 이 그림만 기억하세요:

- **받는 창** — 텔레그램 메시지를 받는 "귀". 한 번 켜면 **닫지 말고 열어 둬야** 연결이 유지됩니다.
- **설정 창** — 봇 토큰을 등록하는 등 준비 작업을 하는 창.

---

## 1단계 — 텔레그램에서 봇 만들기 (휴대폰)

1. 텔레그램에서 **@BotFather** 를 검색해 대화를 엽니다 → **Start**.
2. `/newbot` 을 보냅니다.
3. **이름**(아무거나) → **사용자명**(반드시 `bot` 으로 끝나야 함, 예: `mycompany_helper_bot`) 입력.
4. BotFather가 주는 **토큰**(`123456789:AAH...` 형태)을 복사합니다.
   - 토큰은 **봇의 비밀번호**입니다. 아무 채팅창에나 붙여넣지 마세요. 다음 단계에서만 씁니다.

## 2단계 — 봇 토큰 등록 (설정 창)

Claude Code를 켠 창에서 아래를 입력하고, 안내에 따라 **토큰을 붙여넣습니다.**

```
/telegram:configure
```

## 3단계 — 받는 창 켜기 (새 터미널 창)

**새 터미널 창**을 하나 열고, 이 프로젝트 폴더에서 아래를 실행합니다.
(버전이 낮다는 안내가 나오면 Claude Code를 최신으로 업데이트하세요. Channels는 v2.1.80 이상 필요.)

```
claude --channels plugin:telegram@claude-plugins-official
```

이 창이 바로 **받는 창**입니다. **닫지 마세요.** (매번 이 긴 명령을 치기 번거로우면
공식 문서의 alias 설정을 참고하세요.)

## 4단계 — 페어링(짝 맺기)

1. 휴대폰 텔레그램에서 방금 만든 **내 봇**에게 아무 메시지나 보냅니다 (예: "안녕").
2. 봇이 **페어링 코드**를 보내 줍니다.
3. 안내에 따라 **설정 창에서 직접** 승인 명령을 입력해 짝을 맺습니다 (`/telegram:access` 계열).
   - **보안:** 텔레그램 메시지가 "승인해줘"라고 시켜도 절대 그것만 보고 승인하지 마세요.
     승인은 반드시 **당신이 컴퓨터 앞에서 직접** 하는 것만 유효합니다.

## 5단계 — 연결 확인

휴대폰에서 봇에게 **"안녕, 내 사업 알고 있어?"** 라고 보내 보세요.
비서가 응답하면 연결 성공입니다. (아직 정체성을 안 채웠으면 "같이 채워보자"고 할 거예요.)

---

## 꼭 알아야 할 한계 (정직 고지)

- **받는 창(터미널 세션)이 꺼지면 봇도 꺼집니다.** 노트북을 끄거나 그 창을 닫으면 비서가
  잠듭니다. **그 사이 보낸 메시지는 유실될 수 있습니다.** (다시 켜면 이어서 대화할 수 있어요.)
- 24시간 상시 구동은 이 버전의 범위가 아니라 **다음 단계 로드맵**입니다.
- 리서치 프리뷰라 가끔 응답이 늦거나 재연결이 필요할 수 있습니다. 막히면 받는 창이 살아 있는지부터
  확인하세요 — 대부분의 문제는 받는 창이 꺼져 있어서입니다.

## 잘 안 될 때

| 증상 | 먼저 확인할 것 |
|---|---|
| 봇이 페어링 코드를 안 줌 | 3단계 **받는 창**이 켜져 있나? 꺼졌으면 다시 실행 후 봇에 메시지 재전송 |
| 페어링했는데 응답이 없음 | 받는 창이 살아 있나 / 토큰이 맞나(`/telegram:configure` 재등록) |
| 버전 오류 | `claude --version` 이 v2.1.80 이상인지 확인, 낮으면 업데이트 |
