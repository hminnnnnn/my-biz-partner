# 캘린더 연결하기 (선택 — 폴백 있음)

> 📄 **이 문서는 수동 진행용 백업입니다.** 권장 경로는 터미널에서 `claude` 를 켜고 **"셋업 시작"**
> (또는 "캘린더 연결") 이라고 말해 코치가 폴백을 채워 주고 브리핑을 바로 체험시키는 것입니다.

일정 브리핑을 **내 실제 캘린더 기준**으로 받으려면 구글 캘린더를 연결합니다.
연결이 어렵거나 시간이 없으면 **폴백(수기 일정 파일)** 으로도 똑같이 브리핑을 체험할 수 있으니
게이트 통과가 캘린더 연동에 묶이지 않습니다.

---

## 방법 A — 구글 캘린더 연동

> 연동 방식은 워크숍에서 사용하는 MCP/도구 구성에 따라 조금씩 다릅니다. 진행자가 현장에서
> 최신 절차를 안내합니다. 큰 흐름만 미리 알아 두세요:

1. 구글 캘린더 연동 도구(MCP 등)를 설정합니다.
2. 브라우저에서 구글 계정으로 로그인하고 **캘린더 읽기 권한을 승인**합니다 (OAuth).
3. 승인 후 텔레그램에 **"내일 일정 브리핑해줘"** 라고 하면 실제 캘린더 일정으로 응답합니다.

연동이 되면 파트너는 캘린더를 먼저 봅니다.

---

## 방법 B — 폴백: 수기 일정 파일 (연동 실패·건너뛰기용)

OAuth가 막히거나 시간이 부족해도 괜찮습니다. 아래 파일에 이번 주 일정을 붙여넣으면,
`daily-briefing`(일정 브리핑) 기능이 **캘린더 대신 이 파일을 읽어** 똑같이 브리핑해 줍니다.

**파일 위치:**

```
notes/calendar-fallback.md
```

**하는 법:**

1. `notes/calendar-fallback.md` 를 엽니다 (설치 시 예시가 들어 있습니다).
2. 예시를 지우고 **이번 주 일정을 아무 형식으로나** 적습니다. 예:

   ```
   ## 이번 주 일정
   - 월 10:00 A업체 미팅 (단가 재협의)
   - 화 15:00 B고객 코칭
   - 목 11:00 세무사 통화
   ```

3. 텔레그램에 **"내일 일정 브리핑해줘"** 라고 하면 이 파일 기준으로 답합니다.

> **동작 규약(중요):** 파트너는 브리핑할 때 **① 캘린더 연동이 있으면 캘린더 → ② 없거나 실패하면
> `notes/calendar-fallback.md`** 순서로 읽습니다. 이 규약은 `.claude/skills/daily-briefing/SKILL.md`
> 에도 동일하게 적혀 있습니다. 둘 중 어느 경로든 브리핑 결과는 같은 형식으로 나옵니다.
