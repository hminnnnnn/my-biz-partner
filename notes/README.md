# notes/ 저장 규약 (기록의 기준 문서)

이 폴더는 당신이 텔레그램으로 던진 메모가 **분류되어 쌓이는 곳**입니다.
`quick-note`(저장)와 `recall`(회상) 스킬은 **똑같이 이 규약을 따릅니다.**

## 분류 폴더 (5종)

| 폴더 | 무엇을 넣나 | 예시 |
|---|---|---|
| `inbox/` | 분류가 애매한 기본함. 어디 넣을지 모르면 여기 | "이거 나중에 확인" |
| `meetings/` | 미팅·통화·상담 기록 | "3시 A업체 미팅 내용" |
| `customers/` | 고객·거래처별 기록 | "B고객 요청사항" |
| `ideas/` | 아이디어·떠오른 생각 | "여름 프로모션 아이디어" |
| `issues/` | 이슈·문제 (issue-tracker와 연계) | "반품 클레임" |

**분류가 겹칠 때의 우선순위:** 특정 고객·거래처가 주인공인 기록은 — 첫 문의·상담 통화라도 —
`customers/` 에 넣습니다 (고객 축으로 기록이 쌓여야 나중에 "그 고객 얘기" 로 찾기 쉽습니다).
`meetings/` 는 여러 사람이 얽힌 회의나 사건 중심 기록에 씁니다. 그래도 애매하면 `inbox/`.

> 참고: `projects/` 폴더는 위 5분류와 별개로 **project-team 스킬 전용** 산출물 폴더입니다
> (`projects/<YYYY-MM-프로젝트슬러그>/01-단계.md`). 각 프로젝트 폴더에는 단계별 산출물과 함께
> 완료 시 **`99-최종종합.md`**(최종 종합)와 **`99-회고.md`**(잘된 것/막힌 것/다음에 다르게 — 각 3줄)가
> 남습니다. 이 회고는 다음 프로젝트 착수 때 각 역할이 recall 로 훑어 경험이 축적됩니다.
> quick-note가 분류해 넣는 곳이 아니며, recall은 여기도 함께 검색합니다.

## 파일명 규칙

```
YYYY-MM-DD-짧은제목.md
```

예: `2026-07-20-A업체-단가조정.md`, `2026-07-20-B고객-재계약.md`

- 날짜는 기록한 날. 같은 날 같은 주제가 이어지면 같은 파일에 이어 적습니다.
- 제목은 나중에 찾기 쉬운 핵심어로 짧게.

## 파일 안 형식 (권장)

```markdown
# 짧은제목

- 날짜: 2026-07-20 14:30
- 분류: customers
- 태그: A업체, 단가

본문 — 던진 메모 원문 + 파트너가 정리한 요약.
```

> 이 규약은 `CLAUDE.md`, `.claude/skills/quick-note/SKILL.md`, `.claude/skills/recall/SKILL.md`
> 세 곳에서 동일하게 참조합니다. 경로 표기(`notes/<폴더>/<파일>.md`)를 바꾸지 마세요.
